PRODUCT CONSTITUTION

PERSONAL OPS SERVER

개인 운영을 맡는
AI 전문 비서 시스템

사용자는 자연어로 목적과 자료를 전달하고, 한 명의 주 비서가 맥락을 복원해 프로젝트·일정·지식·결정·후속 조치를 하나의 운영 세계에서 관리합니다.

01

WHY

무엇을 해결하려는가

현재의 문제

정보보다 조율 비용이 더 큽니다.

생각, 회의, 문서, 일정, 약속과 지식이 서로 다른 도구에 흩어지면 사용자가 직접 분류하고 맥락을 복구하며 누락된 후속 조치를 기억해야 합니다.

  • 어디에 저장할지 매번 판단
  • 행동 전에 맥락을 다시 조립
  • 중복·충돌 상태를 수동으로 확인
  • 결정과 약속을 기억에 의존

제품의 약속

사용자는 말하고, 시스템은 운영합니다.

원자료를 자연어로 전달하면 비서가 관련 맥락을 찾고, 필요한 판단만 요청하며, 승인된 일을 안전하게 처리하고 근거와 결과를 남깁니다.

“어디에 기록할지”가 아니라
“무엇을 원하는지”만 말합니다.
02

WORKFLOW

사용자는 짧게 말하고, 비서가 정리합니다

1

자연어로 말하기

정갈한 문서나 입력 양식 없이 실제 상황을 짧고 불완전하게 전달합니다.

2

비서가 한 번 정리

원문을 보존하고 행동·결정·지식·선호·열린 질문과 불확실성을 하나의 메모로 묶습니다.

3

대화로 확인

“저장해”, “월요일로 고쳐줘”, “저장하지 마”처럼 자연어로 승인·교정·거절합니다.

4

받은함에서 검토

승인된 원문과 비서 메모는 대화와 독립되어 남고 변경 이력을 확인할 수 있습니다.

예시

“알파 건은 금요일쯤 민수에게 배포 일정 한번 물어봐야 해.”

→ 알파 프로젝트의 금요일 후속 행동으로 정리하고 저장 여부를 확인합니다.

AI가 판단하는 것

무엇을 메모 후보로 만들 것인가

반드시 제안
기억·저장 요청, 구체적 행동, 결정, 지속 선호, 재사용 지식, 열린 질문
보통 제안하지 않음
인사, 잡담, 답변만 필요하고 지속 가치가 없는 일반 질문
불확실성 유지
“아마”, “금요일쯤” 같은 표현은 확정 사실로 바꾸지 않고 별도 표시
권한 판단
제안은 저장이 아니며 사용자의 후속 자연어 확인 전에는 기준 데이터가 되지 않음

AI에게 제공되는 근거

파일을 탐색하지 않고, 앱이 만든 맥락만 받습니다

  • ai_messages현재 대화의 최근 메시지
  • intake_proposals현재 대화에서 확인을 기다리는 제안
  • assistant_memos최근 확정 메모와 현재 버전
  • assistant_profiles사용자가 확정한 비서 이름·역할·대화 방식·업무 원칙
  • src/ai/streaming-service.ts저장 가치와 확인 방식에 관한 고정 지침
  • src/domain/intake.ts허용되는 출력 종류·길이·구조의 JSON Schema
중요한 경계

웹에서 시작된 AI는 운영체제가 관리하는 저장소 외부의 격리 작업공간에서 실행되며 웹 검색·MCP·하위 Agent·애플리케이션 도메인 도구를 제공받지 않습니다. 개발 저장소의 AGENTS.md, 개인 폴더, WorkOS, SQLite를 직접 읽거나 쓰지 않습니다. src/infra/store.ts가 검증된 결과만 SQLite 트랜잭션으로 적용하고, src/app.ts가 브라우저에 안전한 API만 노출합니다.

RELATIONSHIP

전문 비서의 개념

단일 대화 창구

주 비서는 사용자를 대면하는 하나의 관계입니다.

의도를 해석하고 주의를 보호하며, 필요한 맥락을 모으고, 전문 역할의 결과를 조율해 하나의 운영 답변으로 제시합니다. 대화는 명령과 검토의 표면이지 데이터베이스가 아닙니다.

A
OWNER

사용자

목적, 가치, 원자료, 교정과 중요한 결정의 최종 권한을 제공합니다.

P
PROJECT MANAGER

프로젝트 관리자

목표, 현재 상태, 행동, 회의, 결정, 의존성, 위험과 후속 조치를 추론합니다.

K
KNOWLEDGE RESEARCHER

지식 탐구자

자료를 검색·비교하고 근거와 추론을 분리해 재사용 가능한 지식을 만듭니다.

전문 역할은 별도 채팅방이나 독립 데이터베이스가 아닙니다. 실제로 다른 전문성·도구·맥락·평가가 필요할 때만 주 비서 뒤에 추가됩니다.

AI CONTRACT

AI가 지원해야 하는 것

DO

AI의 책임

  • 의도·주제·긴급성·모호함 파악
  • 목표에 필요한 최소 맥락 검색
  • 프로젝트·일정·사람·결정·지식 연결
  • 필요한 전문 역할에 제한된 위임
  • 근거·상태·추론·제안·미확인 사항 분리
  • 검증 가능한 구조화 작업 의도 반환
  • 누락과 실패를 숨기지 않고 보고
DON'T

AI가 소유하지 않는 것

  • SQLite와 기준 파일 직접 쓰기
  • 일반 셸·무제한 파일시스템 접근
  • CLI 인증정보와 운영체제 비밀
  • 모델 자신이 권한을 확대하는 판단
  • 조용한 삭제·마이그레이션·외부 전송
  • 근거 없는 사실·약속·완료 상태 발명
  • 에이전트별 독립된 운영 데이터
Observe검색·조사·비교·진단
Operate요청된 저위험 작업 적용
Govern삭제·외부 전송 등 별도 승인

TRUSTED WORLD

하나의 공유 운영 세계

01

원자료와 근거

사용자의 원문, 파일, 회의 자료, URL, 시각과 출처를 변형 없이 보존합니다.

02

운영 상태

프로젝트, 약속, 행동, 일정, 사람, 의존성, 위험과 결정의 현재 상태입니다.

03

지식

출처가 있는 주장, 종합, 방법, 모순과 열린 질문을 재사용합니다.

04

비서 기억

제한된 선호와 작업 관례이며 기준 상태를 대신하지 않습니다.

05

감사 기록

작업, 위임, 제안, 승인, 영수증, 교정과 취소 정보를 남깁니다.

08

BOUNDARIES

운영·보안 원칙

한 대의 기준 호스트

집 PC가 기준 저장소이며 서비스는 localhost에만 바인딩됩니다.

Tailnet 전용 접근

PC·갤럭시 탭·스마트폰은 Tailscale Serve로만 연결합니다.

공식 CLI 전용 AI

구독 인증된 Codex·Grok CLI만 사용하며 모델 API 키는 사용하지 않습니다.

WorkOS 독립성

과거 WorkOS는 선택적 참고·가져오기 원천이며 제품 의존성이 아닙니다.
